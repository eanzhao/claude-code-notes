# Claude Code 使用技巧

## 快速导航

| 快捷键 | 功能 |
|--------|------|
| `?` | 显示所有键盘快捷键 |
| `Tab` | 命令补全 |
| `↑` / `↓` | 浏览命令历史 |
| `/` | 显示所有命令和技能 |
| `Ctrl+C` | 中断当前操作 |
| `Ctrl+D` | 退出（在空提示时） |

## 高效提示技巧

### 1. 使用 @ 引用文件

```
@src/auth.js 中的验证逻辑有什么问题？
```

### 2. 使用 ! 执行 bash 命令

```
! npm test
```

### 3. 管道输入

```bash
git diff | claude -p "总结这些更改"
```

### 4. 多行提示

```
请帮我：
1. 读取 src/config.js
2. 添加一个新的配置选项
3. 更新相关文档
4. 运行测试验证
```

```

```

```
将 src/utils/helpers.js 中的函数
```


## 最佳实践

1. **具体说明需求**：越具体，结果越好
2. **提供验证方式**：测试、截图或预期输出
3. **分步执行**：复杂任务拆分成步骤
4. **经常保存**：使用 `/rename` 保存重要会话
5. **清理上下文**：定期使用 `/compact` 或 `/clear`

## 故障排除

### Claude 没有按预期工作

1. 检查 CLAUDE.md 是否被加载：`/memory`
2. 查看当前上下文：`/context`
3. 检查权限设置：`/permissions`
4. 查看模型设置：`/model`

### 响应太慢

1. 切换到更快的模型：`/model sonnet`
2. 减少上下文：使用 `/compact` 或 `/clear`
3. 禁用 MCP 服务器：`/mcp` 然后断开不需要的

### 工具调用失败

1. 检查权限：`/permissions`
2. 查看错误详情：在工具调用输出中
3. 验证环境：使用 `! env` 检查环境变量
